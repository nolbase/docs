# A/B Tests

> Test everything. Ship what wins.

**Module slug:** `abtests`
**Category:** Testing & Growth
**URL:** `/platform/abtests`

## Overview

A/B Tests is NolBase's experimentation framework. It brings statistical rigor to your marketing decisions — test creative variations, audience segments, messaging, and bidding strategies, then let the data tell you what works.

No more guessing. No more "I think this headline is better." Run the test, get the numbers, ship the winner.

## Key Features

### Multi-Variant Testing
Test beyond simple A/B — run A/B/C/D experiments:
- Headlines and copy
- CTAs and button text
- Images and creative assets
- Full page layouts
- Email subject lines and send times
- Ad copy and targeting combinations

### Statistical Significance Engine
Rigorous statistical analysis you can trust:
- **Bayesian analysis** — Probability of each variant being the winner
- **Frequentist analysis** — Traditional confidence intervals and p-values
- Configurable confidence levels (90%, 95%, 99%)
- Minimum detectable effect calculation
- Sample size estimation before you start

### Auto-Winner Selection
Set it and forget it:
- Configure the confidence threshold for declaring a winner
- Winning variant is automatically promoted when the threshold is reached
- Losing variants are deprioritized automatically
- Manual override available if needed

### Traffic Splitting Controls
Fine-tune how traffic is distributed:
- Equal split across variants
- Custom allocation percentages
- Holdout groups for control
- Dynamic reallocation based on early results

### Visual & Copy Testing
Side-by-side comparison tools:
- Preview variants next to each other
- Track engagement metrics per variant in real-time
- Visual heatmap data (where available)

### Detailed Result Reports
Everything you need to make the call:
- Conversion rates per variant
- Confidence intervals with visualization
- Lift calculations (absolute and relative)
- Segment-level breakdowns (device, geography, audience, traffic source)
- Export to PDF or CSV

## How It Works

1. **Hypothesize** — Define what you're testing and why
2. **Create variants** — Build 2–4 variations (or generate with Content Studio)
3. **Configure** — Set traffic split, confidence level, and success metric
4. **Run** — Launch the experiment and let data accumulate
5. **Analyze** — Review results with full statistical breakdown
6. **Ship** — Promote the winner (automatically or manually)
